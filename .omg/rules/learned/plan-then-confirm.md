## กฎ

1. รับ task → บอกแผนเป็น **ตาราง** ก่อน:
   | ลำดับ | ทำอะไร | ผลลัพธ์ (ไฟล์ไหน) | Evidence (ตรวจสอบยังไง) |
2. ระบุชัดเจน: **จะทำ** vs **จะไม่ทำ**
3. รอ user confirm ("ผ่าน"/"ใช่") → ถึงเริ่มทำ
4. ทำทีละขั้น → รายงาน evidence แต่ละขั้น → ไม่ทำทีเดียวจบแล้วค่อยรายงาน
5. ไม่เพิ่ม scope เอง (ไม่ refactor, ไม่ optimize, ไม่สร้างสิ่งที่ไม่ได้ขอ)

## เหตุผล

- User ต้องเห็นล่วงหน้าว่าจะเกิดอะไรขึ้น ไม่ใช่เซอร์ไพรส์
- ป้องกันการทำงานเกิน scope ที่สร้างปัญหาตามมา
- User อ่านโค้ดไม่เป็น → ต้องเห็นแผนเป็นภาษาก่อนตัดสินใจ
