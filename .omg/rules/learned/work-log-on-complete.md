## กฎ

1. จบ task ใด ๆ → อัพเดต `docs/WORK-LOG.md` ทันที
2. Format:
```markdown
## YYYY-MM-DD — <ชื่องาน>

### สิ่งที่ทำ
- ✅ <สิ่งที่เสร็จ> — path หรือไฟล์ที่แตะ
- ✅ <สิ่งที่เสร็จอีก>

### สิ่งที่ยังไม่ทำ
- ❌ <สิ่งที่ยังค้าง> — เหตุผลถ้ามี

### Evidence
- `glob <pattern>` → ผลลัพธ์ที่ตรวจสอบได้
- ไฟล์ที่เปลี่ยนแปลง: <list>
```

3. **สำคัญ:** ไฟล์นี้คือ single source of truth ของความคืบหน้า
4. Session ใหม่ → อ่านไฟล์นี้เป็นอันดับแรก ก่อนทำอะไรทั้งหมด

## เหตุผล

- Agent ไม่มี cross-session memory → ไฟล์นี้คือ "ความจำ" ที่เหลืออยู่
- User ไม่ต้องจำเองว่าทำถึงไหน
- ป้องกันการทำงานซ้ำซ้อนหรือสร้างสิ่งที่ทำไปแล้ว
