# Context Manager Skill

สกิลนี้ทำหน้าที่เป็นหน่วยความจำหลักของโปรเจกต์ ช่วยในการบันทึกและดึงข้อมูลสำคัญเพื่อให้เอเจนต์ทุกตัวทำงานบนฐานข้อมูลเดียวกัน

## Capabilities

- **Keyword Tracking**: บันทึกและอัปเดตคำสำคัญ ชื่อตัวละคร และสถานที่
- **Plot Point Management**: ติดตามเหตุการณ์สำคัญที่เกิดขึ้นแล้วและที่กำลังจะเกิดขึ้น
- **Consistency Check**: ตรวจสอบความสอดคล้องของเนื้อหาใหม่กับบริบทที่บันทึกไว้
- **Context Retrieval**: สรุปบริบทที่จำเป็นสำหรับการเริ่มตอนใหม่หรือการแก้ไขเนื้อหา

## Usage

ใช้สกิลนี้ทุกครั้งที่มีการเปลี่ยนแปลงข้อมูลสำคัญในเรื่อง หรือก่อนเริ่มสร้างเนื้อหาชุดใหม่

### Workflow:

1. **Initialize Context**: สร้างไฟล์ `context.json` ในโฟลเดอร์โปรเจกต์
2. **Update Context**: บันทึกข้อมูลใหม่หลังจบแต่ละตอน (ตอนที่ 1-5, 6-25, ฯลฯ)
3. **Validate Consistency**: ตรวจสอบว่าเนื้อหาใหม่ไม่ขัดแย้งกับ `context.json`

## Context Structure (JSON)

```json
{
  "story_info": {
    "title": "ชื่อเรื่อง",
    "genre": "แนวเรื่อง",
    "total_episodes": 60
  },
  "characters": [
    {
      "name": "ชื่อตัวละคร",
      "role": "บทบาท",
      "traits": ["ลักษณะนิสัย"],
      "current_state": "สถานะปัจจุบัน"
    }
  ],
  "keywords": ["คำสำคัญ 1", "คำสำคัญ 2"],
  "plot_summary": "สรุปเนื้อเรื่องจนถึงปัจจุบัน",
  "last_episode_hook": "จุดดึงดูดจากตอนล่าสุด"
}
```
